FAQ

What is jhttpd?

A Hyper Text Transfer Protocol Daemon written in Java of course! Daemon is actually misleading since it doesn't ever fork() to background, but still...

What can it do?

Basic httpd things, and more! Supports a half-assed creative way of running CGI programs, a stupidly buggy, untested super intuitive file-based configuration format, and plenty of NullPointerExceptions stability!

Bugs? What Bugs?

Remember, if it crashes it's a feature, not a bug!

Anything else?

Oh yeah, it can log stuff too. It's pretty verbose, and Java's Util.logger class is surprisingly difficult to format properly
